# 问题分析报告

> 2026-07-04。设置窗口主题不跟随面板的修复方案,供修复实施用。

---

## 问题:设置页面主题一直黑色,不跟随面板实时更新

### 现象
- 设置窗口一直是深色(黑色),即使面板已切到浅色
- 面板切主题时,设置窗口不实时更新

### 根因

panel 和 settings 是两个独立的 webview 窗口,各自维护 `data-theme` 属性来切换主题。问题出在两者没有同步机制:

1. **SettingsApp 只在挂载时读一次后端 theme**(且当前还保留着主题 section 自己设主题,与面板不同步)。首次打开 settings 时读后端 `theme` 应用 `data-theme`,之后就不再更新。

2. **问题 2 修复后 settings 窗口点 X 改为 hide(不销毁)**,所以 hide→show 不触发 React 组件重新挂载,`useEffect` 不重跑,主题永远停在首次挂载时的值。

3. **面板切主题时只写后端 `setSetting("theme", ...)`,没有广播事件**——settings 窗口无从得知主题变了。

4. **对比**:强调色有 `accent-changed` 事件广播([SettingsApp.tsx](app/src/components/SettingsApp.tsx) emit → [App.tsx](app/src/App.tsx) listen),面板能实时跟随;主题缺少对应的广播机制。

### 解决方法

总体思路:模仿强调色的 `accent-changed` 机制,给主题加一个 `theme-changed` 跨窗口事件。面板切主题时广播,settings 窗口监听并实时更新 `data-theme`。同时设置页不再自己设主题(删主题 section),只被动跟随面板。

#### 1. 面板切主题时广播 `theme-changed`

[App.tsx](app/src/App.tsx) 顶部 import 加 `emit`(已有 `listen`,合并):
```tsx
import { emit, listen } from "@tauri-apps/api/event";
```

[App.tsx](app/src/App.tsx) `handleToggleTheme`(第 74-92 行)末尾加一行广播。`next` 是切换后的主题(`"light"` / `"dark"`),在 `setTheme` 之后 emit:
```tsx
function handleToggleTheme(e: MouseEvent<HTMLButtonElement>) {
  const rect = e.currentTarget.getBoundingClientRect();
  const x = rect.left + rect.width / 2;
  const y = rect.top + rect.height / 2;
  document.documentElement.style.setProperty("--reveal-x", `${x}px`);
  document.documentElement.style.setProperty("--reveal-y", `${y}px`);
  const next: ThemeMode = mode === "dark" ? "light" : "dark";
  const reduce = window.matchMedia("(prefers-reduced-motion: reduce)").matches;
  const doc = document as Document & {
    startViewTransition?: (cb: () => void) => unknown;
  };
  if (doc.startViewTransition && !reduce) {
    doc.startViewTransition(() => {
      flushSync(() => setTheme(next));
    });
  } else {
    setTheme(next);
  }
  // 广播主题变更,settings 窗口实时跟随
  emit("theme-changed", next).catch((e) =>
    console.error("emit theme-changed failed", e),
  );
}
```

#### 2. SettingsApp 删主题 section + 改为被动跟随

[SettingsApp.tsx](app/src/components/SettingsApp.tsx) 当前还保留着主题 section(THEME_OPTIONS、theme state、handleThemeChange、跟随系统 useEffect、主题 JSX)。这些全部删除——设置页不再自己设主题,改为跟随面板。

**删除**:
- `THEME_OPTIONS` 常量
- `theme` state(`const [theme, setTheme] = useState<ThemeValue>(...)`)
- `handleThemeChange` 函数
- 跟随系统的 `useEffect`(`if (theme !== "follow") return; ... mq.addEventListener(...)`)
- 挂载 useEffect 里 `setTheme(t)` / `setAccent(a)` 中 setTheme 相关(保留读 accent 等)
- JSX 的"主题"section(含 `THEME_OPTIONS.map(...)` 那段)

**保留** `resolveTheme` 工具函数(挂载时解析后端 theme 用),但调整签名接受 `undefined`:
```tsx
function resolveTheme(value: string | undefined): "light" | "dark" {
  if (value === "light") return "light";
  if (value === "dark") return "dark";
  // follow 或未设置:跟随系统
  return window.matchMedia("(prefers-color-scheme: dark)").matches ? "dark" : "light";
}
```

**新增**:挂载时读后端 theme 应用 + 监听 `theme-changed` 实时更新。import 加 `listen`、`isTauri`:
```tsx
import { emit } from "@tauri-apps/api/event";
import { listen } from "@tauri-apps/api/event";
import { isTauri } from "@tauri-apps/api/core";
import { getSettings, setSetting, setAutostart } from "../lib/tauri";
```
(若 `emit` 不再被 SettingsApp 使用可只 import `listen`。)

挂载 `useEffect` 改成:
```tsx
useEffect(() => {
  if (!isTauri()) return;
  // 首次:读后端当前 theme 应用 data-theme
  getSettings()
    .then((s) => {
      document.documentElement.setAttribute("data-theme", resolveTheme(s.theme));
    })
    .catch((e) => console.error("load theme failed", e));
  // 实时:监听面板广播的 theme-changed
  const un = listen<string>("theme-changed", (e) => {
    const v = e.payload;
    if (v === "light" || v === "dark") {
      document.documentElement.setAttribute("data-theme", v);
    }
  });
  return () => {
    un.then((f) => f());
  };
}, []);
```

注意:原有的 accent / maxItems / autostart 读取逻辑保留(可在同一个 useEffect 里一起读,或分开)。主题不再有 state,直接写 `document.documentElement` 即可。

#### 3. 权限确认

settings 窗口的 capability [capabilities/settings.json](app/src-tauri/capabilities/settings.json) 已含 `core:event:default`,能 `listen` 跨窗口事件;panel 的 [capabilities/default.json](app/src-tauri/capabilities/default.json) 同样有 `core:event:default`,能 `emit`。无需改动。

`emit("theme-changed", next)` 会广播给所有窗口(含 panel 自己),但 panel 不 listen 这个事件,所以无副作用;settings listen 后实时更新。
